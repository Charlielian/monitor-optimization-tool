# 告警过滤功能测试示例

## 🧪 测试用例

### 测试1：单个网元ID
```
输入框：12635595
预期SQL：WHERE ne_id LIKE '%12635595%'
预期日志：├─ 过滤网元ID: 12635595
```

### 测试2：多个网元ID（逗号分隔）
```
输入框：12635595,12635596,12635597
预期SQL：WHERE (ne_id LIKE '%12635595%' OR ne_id LIKE '%12635596%' OR ne_id LIKE '%12635597%')
预期日志：
├─ 过滤网元ID: 12635595
├─ 过滤网元ID: 12635596
├─ 过滤网元ID: 12635597
```

### 测试3：网元ID + 站点名称（空格分隔）
```
输入框：12635595 北京站点
预期SQL：WHERE (ne_id LIKE '%12635595%' AND site_name LIKE '%北京站点%')
预期日志：├─ 过滤条件: 网元ID=12635595, 站点名称=北京站点
```

### 测试4：混合过滤
```
输入框：12635595 北京,12635596,12635597 上海
预期SQL：WHERE (
    (ne_id LIKE '%12635595%' AND site_name LIKE '%北京%') OR
    (ne_id LIKE '%12635596%') OR
    (ne_id LIKE '%12635597%' AND site_name LIKE '%上海%')
)
预期日志：
├─ 过滤条件: 网元ID=12635595, 站点名称=北京
├─ 过滤网元ID: 12635596
├─ 过滤条件: 网元ID=12635597, 站点名称=上海
```

### 测试5：站点名称包含空格
```
输入框：12635595 北京 朝阳区
预期SQL：WHERE (ne_id LIKE '%12635595%' AND site_name LIKE '%北京 朝阳区%')
预期日志：├─ 过滤条件: 网元ID=12635595, 站点名称=北京 朝阳区
```

### 测试6：逗号前后有空格
```
输入框：12635595 北京 , 12635596 上海
预期：自动去除逗号前后空格，等同于"12635595 北京,12635596 上海"
```

## 🐛 常见问题排查

### 问题1：输入后没有结果
**排查步骤**：
1. 查看日志中的过滤条件是否正确解析
2. 尝试只输入网元ID，看是否有结果
3. 检查数据库中是否存在该网元ID和站点名称

### 问题2：日志中没有显示过滤条件
**排查步骤**：
1. 确认日志级别设置为INFO或DEBUG
2. 检查日志文件路径是否正确
3. 尝试重启应用

### 问题3：查询很慢
**排查步骤**：
1. 查看日志中的SQL查询耗时
2. 检查数据库索引是否创建成功
3. 运行 `SHOW INDEX FROM cur_alarm;` 查看索引

### 问题4：站点名称过滤不生效
**排查步骤**：
1. 确认输入格式：网元ID 空格 站点名称
2. 检查数据库中site_name字段是否有值
3. 尝试使用站点名称的一部分（模糊匹配）
