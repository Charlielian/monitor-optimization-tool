# 网页监控 Flask 应用使用说明

## ⚠️ 重要提示

**不要直接打开 HTML 文件！** 必须通过 Flask 应用访问。

## 正确的访问方式

### 1. 启动 Flask 应用

在项目目录下运行：

```bash
# 激活虚拟环境（如果使用）
source venv/bin/activate

# 启动应用
python3 app.py
```

或者指定端口：

```bash
python3 app.py --port 5001
```

### 2. 在浏览器中访问

启动后，在浏览器中访问：

```
http://127.0.0.1:5000/
```

或

```
http://localhost:5000/
```

**不要**直接打开 `templates/base.html` 或任何模板文件！

## 静态资源说明

### 已下载的资源

静态资源文件已下载到以下位置：
- `static/css/bootstrap.min.css`
- `static/js/bootstrap.bundle.min.js`
- `static/js/chart.umd.min.js`

### 验证资源是否存在

运行以下命令检查：

```bash
ls -la static/css/bootstrap.min.css
ls -la static/js/bootstrap.bundle.min.js
ls -la static/js/chart.umd.min.js
```

如果文件不存在，运行：

```bash
python3 download_static_resources.py
```

## 常见问题

### 1. 页面显示警告"Chart.js 未加载"

**可能原因：**
- 直接打开了 HTML 文件（file:// 协议）
- Flask 应用未运行
- 静态资源文件不存在

**解决方法：**
1. 确保 Flask 应用正在运行
2. 通过 `http://127.0.0.1:5000/` 访问，而不是直接打开文件
3. 检查静态资源文件是否存在

### 2. 页面样式异常

**可能原因：**
- Bootstrap CSS 未加载
- 直接打开了 HTML 文件

**解决方法：**
1. 确保通过 Flask 应用访问
2. 检查浏览器控制台是否有错误信息

### 3. 图表不显示

**可能原因：**
- Chart.js 未加载
- 数据为空
- JavaScript 错误

**解决方法：**
1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签页的错误信息
3. 查看 Network 标签页，确认资源是否成功加载

## 调试步骤

1. **检查 Flask 应用是否运行**
   ```bash
   ps aux | grep python | grep app.py
   ```

2. **检查静态资源文件**
   ```bash
   ls -la static/css/
   ls -la static/js/
   ```

3. **查看浏览器控制台**
   - 按 F12 打开开发者工具
   - 查看 Console 标签页的错误信息
   - 查看 Network 标签页，确认资源加载状态

4. **检查 Flask 日志**
   - 查看终端输出的日志信息
   - 检查 `logs/monitoring_app.log` 文件

2. ✅ 在浏览器访问：`http://127.0.0.1:5000/`
3. ❌ **不要**直接打开 HTML 文件
