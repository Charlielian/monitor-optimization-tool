# 用户管理功能说明

## 功能概述

管理员可以在管理员控制台中完整管理所有系统用户，包括：
- ✓ 添加新用户
- ✓ 删除用户
- ✓ 修改用户密码
- ✓ 查看用户列表
- ✓ 配置持久化保存

## 访问方式

1. 使用管理员账号登录系统
2. 点击导航栏的"管理员"菜单
3. 在"用户管理"卡片中进行操作

## 功能详解

### 1. 查看用户列表

用户列表显示所有系统用户的信息：
- 用户名（唯一标识）
- 姓名（显示名称）
- 角色（管理员/普通用户）
- 最后登录时间
- 操作按钮

### 2. 添加新用户

**操作步骤：**
1. 点击"添加用户"按钮
2. 填写用户信息：
   - 用户名：只能包含字母、数字和下划线
   - 姓名：用户的显示名称
   - 密码：至少6个字符
   - 角色：选择"普通用户"或"管理员"
3. 点击"添加"按钮

**验证规则：**
- 用户名不能重复
- 用户名只能包含字母、数字和下划线
- 密码长度至少6个字符
- 所有字段必填

**示例：**
```
用户名: zhangsan
姓名: 张三
密码: zhang123
角色: 普通用户
```

### 3. 修改用户密码

**操作步骤：**
1. 在用户列表中找到目标用户
2. 点击"编辑"按钮（铅笔图标）
3. 在弹出的对话框中：
   - 用户名自动填充（不可修改）
   - 输入新密码（至少6个字符）
   - 确认新密码（必须与新密码一致）
4. 点击"修改"按钮

**验证规则：**
- 新密码长度至少6个字符
- 两次输入的密码必须一致
- 密码会自动加密存储（scrypt算法）

**安全特性：**
- 密码使用scrypt算法加密存储
- 不会显示原密码
- 修改后立即生效
- 操作会记录到系统日志

### 4. 删除用户

**操作步骤：**
1. 在用户列表中找到目标用户
2. 点击"删除"按钮（垃圾桶图标）
3. 确认删除操作

**限制条件：**
- 不能删除当前登录的用户
- 删除操作不可恢复
- 需要二次确认

**注意事项：**
- 删除用户后，该用户将无法登录
- 用户的历史操作记录不会被删除
- 建议在删除前确认用户不再需要访问系统

### 2. 配置持久化

用户信息保存在 `config.json` 文件中：

```json
{
    "auth_config": {
        "enable_auth": true,
        "session_lifetime_hours": 24,
        "users": {
            "admin": {
                "password_hash": "scrypt:32768:8:1$...",
                "role": "admin",
                "name": "系统管理员"
            },
            "user": {
                "password_hash": "scrypt:32768:8:1$...",
                "role": "user",
                "name": "普通用户"
            }
        }
    }
}
```

### 3. 密码加密

使用 Werkzeug 的 scrypt 算法：
```python
from werkzeug.security import generate_password_hash, check_password_hash

# 生成密码哈希
password_hash = generate_password_hash("password123")

# 验证密码
is_valid = check_password_hash(password_hash, "password123")
```

**scrypt 参数：**
- N=32768 (CPU/内存成本)
- r=8 (块大小)
- p=1 (并行度)

  var newPassword = document.querySelector('input[name="new_password"]').value;
  var confirmPassword = document.getElementById('confirm_password').value;

3. 点击"添加用户"

## 安全特性

### 1. 权限控制
- 只有管理员可以访问用户管理功能
- 使用 `@admin_required` 装饰器保护路由
- 非管理员访问会被重定向

### 2. 密码安全
- 密码使用 scrypt 算法加密
- 不存储明文密码
- 密码哈希包含随机盐值
- 每次修改密码都会生成新的哈希

### 3. 操作日志
所有用户管理操作都会记录到系统日志：
```
2025-12-29 10:00:00 - INFO - 管理员 admin 添加了新用户: zhangsan
2025-12-29 11:00:00 - INFO - 管理员 admin 修改了用户 zhangsan 的密码
2025-12-29 12:00:00 - INFO - 管理员 admin 删除了用户: zhangsan
```

### 4. 输入验证
- 前端：HTML5 表单验证
- 后端：Python 参数验证
- 防止SQL注入（使用配置文件，不涉及数据库）
- 防止XSS攻击（Flask自动转义）

### 5. 自我保护
- 管理员不能删除自己的账号
- 防止系统无管理员的情况
- 建议至少保留一个管理员账号

## 注意事项

### 1. 配置文件备份
在进行用户管理操作前，建议备份 `config.json`：
```bash
cp config.json config.json.backup
```

### 2. 多管理员协作
- 多个管理员可以同时管理用户
- 配置文件会实时更新
- 建议协调操作，避免冲突

### 3. 密码策略
当前密码要求：
- 最小长度：6个字符
- 无复杂度要求

建议的密码策略：
- 长度至少8个字符
- 包含大小写字母、数字、特殊字符
- 定期更换密码
- 不使用常见密码

### 4. 用户数量限制
- 当前无用户数量限制
- 建议根据实际需求控制用户数量
- 定期清理不活跃用户

### 5. 配置文件权限
确保 `config.json` 文件权限正确：
```bash
chmod 600 config.json  # 只有所有者可读写
```

## 故障排除

### 问题1：添加用户失败
**可能原因：**
- 用户名已存在
- 配置文件权限不足
- 配置文件格式错误

**解决方法：**
1. 检查用户名是否重复
2. 检查 config.json 文件权限
3. 验证 config.json 格式是否正确

### 问题2：删除用户失败
**可能原因：**
- 尝试删除当前登录用户
- 配置文件权限不足

**解决方法：**
1. 使用其他管理员账号删除
2. 检查文件权限

### 问题3：修改密码后无法登录
**可能原因：**
- 密码输入错误
- 配置未正确保存

**解决方法：**
1. 确认新密码
2. 检查 config.json 中的密码哈希是否更新
3. 重启应用

### 问题4：配置文件损坏
**解决方法：**
1. 恢复备份文件
2. 手动修复 JSON 格式
3. 使用 config.example.json 作为模板
