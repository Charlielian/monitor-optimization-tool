## 如何使用这些日志定位性能问题

1. **查看总耗时**：找到 "告警页面总耗时" 日志
2. **定位慢查询**：查找带 ⚠️ 或 🔴 标记的日志
3. **分析瓶颈**：
   - 如果 "当前告警查询" 慢 → 检查 cur_alarm 表的索引
   - 如果 "历史告警查询" 慢 → 检查时间范围和去重逻辑
   - 如果 "告警统计" 慢 → 考虑添加缓存
   - 如果 "模板渲染" 慢 → 检查数据量和模板复杂度

## 优化建议

### 数据库优化
```sql
-- 为 cur_alarm 表添加索引
CREATE INDEX idx_import_time ON cur_alarm(import_time);
CREATE INDEX idx_ne_id ON cur_alarm(ne_id);
CREATE INDEX idx_alarm_level ON cur_alarm(alarm_level);
CREATE INDEX idx_occur_time ON cur_alarm(occur_time);

-- 复合索引（用于历史告警查询）
CREATE INDEX idx_import_time_ne_id ON cur_alarm(import_time, ne_id);
```

### 应用层优化
1. **添加缓存**：对告警统计数据使用1分钟缓存
2. **分页优化**：减少每页显示数量（当前100条）
3. **异步加载**：考虑将统计数据和列表数据分开加载
4. **查询优化**：避免使用 SELECT *，只查询需要的字段
