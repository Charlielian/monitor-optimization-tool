# 告警监控查询功能说明

## 功能概述

告警监控查询功能提供了对MySQL数据库中`cur_alarm`表的查询和展示，包括当前告警和历史告警两个子页面。

## 数据库配置

### MySQL连接信息

系统会自动从 `config.json` 文件中读取MySQL配置：

```json
{
  "mysql_config": {
    "host": "192.168.31.175",
    "port": 3306,
    "database": "optimization_toolbox",
    "user": "root",
    "password": "103001"
  }
}
```

### 数据表
- **表名**: `cur_alarm`
- **用途**: 存储告警数据

## 功能特性

### 1. 当前告警

**查询逻辑**：
- 当前时间在0点0分后：查询23:30-0:00的告警
- 当前时间在0点30分后：查询00:00-0:30的告警
- 如果查询到比计算时间更新的数据，则使用最新的数据

**显示内容**：
- 告警时间
- 告警级别（紧急/重要/一般）
- 告警名称
- 告警类型
- 小区名称
- 小区ID
- 制式（4G/5G）
- 告警来源
- 告警状态
- 告警描述

### 2. 历史告警

**查询逻辑**：
- 支持自定义时间范围查询
- 默认查询最近7天的数据
- 剔除`import_time`、`import_filename`、`import_batch`三列后进行去重
- 支持分页显示（每页100条）

**显示内容**：
与当前告警相同，但数据已去重

### 3. 告警统计

在页面顶部显示4个统计卡片：
- **当前告警总数**: 最近半小时的告警数量
- **紧急告警**: 当前紧急级别的告警数量
- **重要告警**: 当前重要级别的告警数量
- **今日告警总数**: 今天的告警总数（去重）

### 4. 数据导出

支持将当前告警和历史告警导出为Excel文件：
- **当前告警导出**: 导出最近半小时的所有告警
- **历史告警导出**: 导出查询时间范围内的所有告警（去重后）
- Excel文件包含中文表头和样式美化

## 使用方法

### 1. 访问告警监控页面

启动Flask应用后，在导航栏点击"告警监控"，或直接访问：
```
http://127.0.0.1:5000/alarm
```

### 2. 查看当前告警

- 默认显示"当前告警"标签页
- 自动查询最近半小时的告警数据
- 点击"刷新"按钮可更新数据
- 点击"导出Excel"可下载当前告警数据

### 3. 查询历史告警

- 切换到"历史告警"标签页
- 选择开始时间和结束时间
- 点击"查询"按钮
- 支持分页浏览
- 点击"导出Excel"可下载历史告警数据

### 4. 测试告警服务

运行测试脚本验证功能：
```bash
python test_alarm_service.py
```

测试内容包括：
- MySQL连接测试
- 告警统计查询
- 当前告警查询
- 历史告警查询（带分页）

## API接口

### 告警监控页面
- **路由**: `/alarm`
- **方法**: GET
- **参数**:
  - `tab`: 标签页（current/historical）
  - `start_time`: 开始时间（历史告警）
  - `end_time`: 结束时间（历史告警）
  - `page`: 页码（历史告警）

### 导出当前告警
- **路由**: `/export/current_alarms.xlsx`
- **方法**: GET
- **返回**: Excel文件

### 导出历史告警
- **路由**: `/export/historical_alarms.xlsx`
- **方法**: GET
- **参数**:
  - `start_time`: 开始时间
  - `end_time`: 结束时间
- **返回**: Excel文件

## 注意事项

1. **数据库连接**: 确保MySQL服务正常运行，配置信息正确
2. **表结构**: 确保`cur_alarm`表存在且包含必要的字段
3. **性能优化**: 历史告警查询使用了DISTINCT去重，大数据量时可能较慢
4. **时间格式**: 页面使用HTML5的datetime-local输入控件
5. **权限控制**: 如果启用了认证，需要登录后才能访问

## 故障排查

### 问题1: 告警服务未初始化
**原因**: MySQL连接失败
**解决**: 检查config.json中的mysql_config配置

### 问题2: 查询不到数据
**原因**: 
- 数据库中没有数据
- 时间范围不正确
- 表名或字段名不匹配

**解决**: 
- 运行test_alarm_service.py测试
- 检查数据库中是否有数据
- 确认表结构是否正确

### 问题3: 导出失败
**原因**: 数据量过大或权限问题
**解决**: 
- 缩小时间范围
- 检查文件写入权限
- 查看日志文件
