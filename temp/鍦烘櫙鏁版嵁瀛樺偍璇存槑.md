# 场景数据存储说明

## 存储位置

场景数据保存在 **PostgreSQL 数据库**中。

## 数据库配置

数据库连接信息在 `config.json` 文件中配置：

```json
{
    "pgsql_config": {
        "host": "188.15.68.62",
        "port": 54326,
        "database": "data_wg",
        "user": "postgres",
        "password": "10300"
    }
}
```

## 数据表结构

### 1. `scenarios` 表 - 场景基本信息

存储场景的名称、描述等信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PRIMARY KEY | 场景ID（自增） |
| scenario_name | TEXT UNIQUE NOT NULL | 场景名称（唯一） |
| description | TEXT | 场景描述 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

**创建语句：**
```sql
CREATE TABLE IF NOT EXISTS scenarios (
    id SERIAL PRIMARY KEY,
    scenario_name TEXT UNIQUE NOT NULL,
    description TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### 2. `scenario_cells` 表 - 场景下的小区信息

存储每个场景包含的小区列表。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PRIMARY KEY | 记录ID（自增） |
| scenario_id | INTEGER NOT NULL | 场景ID（外键，关联 scenarios.id） |
| cell_id | TEXT NOT NULL | 小区ID |
| cell_name | TEXT | 小区名称 |
| cgi | TEXT | CGI标识 |
| network_type | TEXT NOT NULL | 网络制式（4G/5G） |
| created_at | TIMESTAMP | 创建时间 |

**唯一约束：** `(scenario_id, cell_id, network_type)` - 同一场景下，同一小区ID和制式的组合只能出现一次。

**创建语句：**
```sql
CREATE TABLE IF NOT EXISTS scenario_cells (
    id SERIAL PRIMARY KEY,
    scenario_id INTEGER NOT NULL REFERENCES scenarios(id) ON DELETE CASCADE,
    cell_id TEXT NOT NULL,
    cell_name TEXT,
    cgi TEXT,
    network_type TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(scenario_id, cell_id, network_type)
);
```

## 数据管理

### 自动创建表

应用启动时，`ScenarioService` 会自动检查并创建这两个表（如果不存在）。

代码位置：`services/scenario_service.py` 的 `_ensure_tables()` 方法。

### 数据操作

所有场景相关的数据操作都通过 `ScenarioService` 类进行：

- **创建场景**：`create_scenario(name, desc)`
- **删除场景**：`delete_scenario(scenario_id)` - 级联删除关联的小区
- **列出场景**：`list_scenarios()`
- **添加小区**：`add_cells(scenario_id, cells)`
- **移除小区**：`remove_cell(scenario_id, cell_id, network_type)`
- **列出小区**：`list_cells(scenario_id)`

## 数据查询

### 查看所有场景

```sql
SELECT id, scenario_name, description, created_at, updated_at 
FROM scenarios 
ORDER BY scenario_name;
```

### 查看场景下的小区

```sql
SELECT id, cell_id, cell_name, cgi, network_type
FROM scenario_cells
WHERE scenario_id = 1
ORDER BY network_type, cell_id;
```

### 查看场景及其小区数量

```sql
SELECT 
    s.id,
    s.scenario_name,
    COUNT(sc.id) as cell_count
FROM scenarios s
LEFT JOIN scenario_cells sc ON s.id = sc.scenario_id
GROUP BY s.id, s.scenario_name
ORDER BY s.scenario_name;
```

## 数据备份

### 导出场景数据

```bash
# 导出场景表
pg_dump -h 188.15.68.62 -p 54326 -U postgres -d data_wg -t scenarios > scenarios_backup.sql

# 导出场景小区表
pg_dump -h 188.15.68.62 -p 54326 -U postgres -d data_wg -t scenario_cells > scenario_cells_backup.sql
```

### 导入场景数据

```bash
# 导入场景表
psql -h 188.15.68.62 -p 54326 -U postgres -d data_wg < scenarios_backup.sql

# 导入场景小区表
psql -h 188.15.68.62 -p 54326 -U postgres -d data_wg < scenario_cells_backup.sql
```

## 注意事项

1. **级联删除**：删除场景时，会自动删除该场景下的所有小区记录（`ON DELETE CASCADE`）
2. **唯一性约束**：同一场景下，同一小区ID和制式的组合不能重复
3. **数据持久化**：所有场景数据都保存在数据库中，重启应用不会丢失
4. **数据库连接**：确保数据库服务正常运行，否则应用无法访问场景数据

## 相关文件

- **服务类**：`services/scenario_service.py` - 场景数据操作逻辑
- **数据库客户端**：`db/pg.py` - PostgreSQL 连接和操作
- **配置文件**：`config.json` - 数据库连接配置
- **应用入口**：`app.py` - Flask 应用，使用 ScenarioService
