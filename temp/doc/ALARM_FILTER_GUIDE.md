# 告警过滤功能使用指南

## 🎯 网元ID / 站点名称过滤功能

### 功能说明
在告警查询页面的"网元ID"输入框中，支持以下三种过滤方式：

### 1️⃣ 单个网元ID过滤
**输入格式**：直接输入网元ID
```
12635595
```

**效果**：查询包含该网元ID的所有告警（模糊匹配）

---

### 2️⃣ 多个网元ID过滤
**输入格式**：多个网元ID用英文逗号分隔
```
12635595,12635596,12635597
```

**效果**：查询包含任意一个网元ID的告警（OR逻辑）

---

### 3️⃣ 网元ID + 站点名称组合过滤 ⭐ 新功能
**输入格式**：网元ID 空格 站点名称
```
12635595 北京站点
```

**效果**：同时匹配网元ID和站点名称（AND逻辑，都要满足）

---

### 4️⃣ 混合过滤（组合使用）
**输入格式**：用逗号分隔多个条件，每个条件可以是"网元ID"或"网元ID 站点名称"
```
12635595 北京站点,12635596,12635597 上海站点
```

**效果**：
- 第1个条件：网元ID包含"12635595" **且** 站点名称包含"北京站点"
- 第2个条件：网元ID包含"12635596"
- 第3个条件：网元ID包含"12635597" **且** 站点名称包含"上海站点"
- 三个条件之间是 **OR** 关系（满足任意一个即可）

## 🔍 模糊匹配说明

所有过滤都支持**模糊匹配**（LIKE查询）：

### 网元ID模糊匹配
```
输入：1263
结果：匹配 12635595, 12635596, 12630001 等所有包含"1263"的网元ID
```

### 站点名称模糊匹配
```
输入：12635595 北京
结果：匹配站点名称包含"北京"的，如：
- 北京站点A
- 北京市朝阳区站点
- 北京东城站
```

## ⚠️ 注意事项

### 1. 空格的作用
- **第一个空格**：用于分隔网元ID和站点名称
- **后续空格**：作为站点名称的一部分

```
示例：12635595 北京 朝阳区
解析为：
- 网元ID：12635595
- 站点名称：北京 朝阳区（包含空格）
```

### 2. 逗号的作用
- 用于分隔多个过滤条件
- 逗号前后的空格会被自动去除

```
示例：12635595 北京 , 12635596 上海
等同于：12635595 北京,12635596 上海
```

### 3. 大小写
- 查询不区分大小写（MySQL默认）
- 建议使用实际的大小写格式

### 4. 特殊字符
- 避免使用 `%` 和 `_`（SQL通配符）
- 如需查询包含这些字符的内容，请联系管理员

## 📊 SQL查询逻辑

### 单个网元ID
```sql
WHERE ne_id LIKE '%12635595%'
```

### 多个网元ID
```sql
WHERE (ne_id LIKE '%12635595%' OR ne_id LIKE '%12635596%')
```

### 网元ID + 站点名称
```sql
WHERE (ne_id LIKE '%12635595%' AND site_name LIKE '%北京%')
```

### 混合条件
```sql
WHERE (
    (ne_id LIKE '%12635595%' AND site_name LIKE '%北京%') OR
    (ne_id LIKE '%12635596%') OR
    (ne_id LIKE '%12635597%' AND site_name LIKE '%上海%')
)
```

## 🐛 常见问题

### Q1: 输入了网元ID和站点名称，但没有结果？
**A**: 检查以下几点：
1. 网元ID和站点名称之间是否有空格
2. 网元ID和站点名称是否都正确
3. 尝试只输入网元ID，看是否有结果
4. 尝试只输入站点名称的一部分（如"北京"而不是"北京朝阳站点A"）

### Q2: 如何查询所有北京的告警？
**A**: 只输入站点名称不支持，需要配合网元ID：
```
方法1：输入所有北京的网元ID（用逗号分隔）
方法2：使用告警名称过滤（如果告警名称包含地区信息）
```

### Q3: 空格输入错了怎么办？
**A**: 
- 如果多输入了空格，站点名称会包含空格，可能匹配不到
- 建议：网元ID和站点名称之间只用一个空格

### Q4: 逗号输入成中文逗号了？
**A**: 
- 必须使用英文逗号 `,`
- 中文逗号 `，` 会被当作站点名称的一部分
